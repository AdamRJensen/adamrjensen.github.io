Retrieving MERRA-2 data#

MERRA-2 is the global reanalysis dataset produced by NASA and contains a lot of interesting atmospheric data relevant to modeling solar radiation. Particularly, the hourly aerosol data are of interest. More clasical variables such as temperature and wind speed are also avaiable.

In this post will deal with MERRA-2 Hourly 2-Dimensional Time-Averaged Variables (M2T1NXSLV) and how to retrieve and manipulate it.

First, off it should be mentioned that retrieval of MERRA-2 data is not a simple task! The hourly data is stored in daily NetCDF files covering the entire world at GES DISC).

There are three main methods of getting subsetted data (not considering user interfaces):

  • Manually construct the filenames based on start and end time

  • Using OpeNDAP

  • GES DISC subsetting API

The advantage of the two latter methods is that they automatically uses the newest stream number, e.g., 401 for September 2020.

def _merra_stream(d):
    if (d.year == 2020) & (d.month == 9):
        # Due to an error, MERRA-2 data has been reprocessed for September
        # 2020 and the reprocessed/corrected data file is version 401
        return 401
    elif d.year >= 2011:
        return 400
    elif (d.year >= 2001) & (d.year < 2011):
        return 300
    elif (d.year >= 1992) & (d.year < 2001):
        return 200
    elif (d.year >= 1980) & (d.year < 1992):
        return 100

        raise ValueError('Requested period cannot be before 1980.')
